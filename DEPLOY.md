# Quick Deploy Guide

## Deploy to Render via GitHub

### Step 1: Push to GitHub
```bash
git add .
git commit -m "Add Docker files for Render deployment"
git push origin main
```

### Step 2: Deploy on Render

#### Option A: Using Blueprint (Recommended)
1. Go to https://dashboard.render.com
2. Click "New +" â†’ "Blueprint"
3. Connect your GitHub repository
4. Render will automatically detect `render.yaml` and create all services
5. Configure environment variables (see below)
6. Deploy!

#### Option B: Manual Setup
1. Create PostgreSQL database first
2. Create backend web service
3. Create frontend web service
4. Configure environment variables

### Step 3: Environment Variables

#### Backend (`fortnite-course-backend`):
- `DATABASE_URL` - Auto-set if using Render PostgreSQL
- `SECRET_KEY` - Auto-generated by Render
- `FRONTEND_URL` - Auto-set from frontend service
- `CORS_ORIGINS` - Auto-set from frontend service
- `SMTP_USER` - Your email (for password reset)
- `SMTP_PASSWORD` - Your email app password
- `SMTP_FROM_EMAIL` - Your email address
- `GOOGLE_CLIENT_ID` - (Optional) Google OAuth
- `GOOGLE_CLIENT_SECRET` - (Optional) Google OAuth

#### Frontend (`fortnite-course-frontend`):
- `NEXT_PUBLIC_API_URL` - Auto-set from backend service
- `NODE_ENV` - Auto-set to "production"

### Step 4: Initialize Database

After deployment, run migrations:
```bash
# Using Render Shell
cd backend
alembic upgrade head
python scripts/init_db.py
```

### Step 5: Access Your App

- Frontend: `https://fortnite-course-frontend.onrender.com`
- Backend API: `https://fortnite-course-backend.onrender.com`
- Health Check: `https://fortnite-course-backend.onrender.com/health`

## Local Development

```bash
# Start all services
docker-compose up -d

# View logs
docker-compose logs -f

# Stop services
docker-compose down
```

## Troubleshooting

- **Backend won't start**: Check `DATABASE_URL` is correct
- **Frontend can't connect**: Verify `NEXT_PUBLIC_API_URL` points to backend
- **CORS errors**: Ensure `CORS_ORIGINS` includes frontend URL
- **Database connection**: Use Internal Database URL from Render dashboard
